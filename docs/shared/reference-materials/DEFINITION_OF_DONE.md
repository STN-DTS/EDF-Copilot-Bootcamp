# Definition of Done (Bootcamp Standard)

This checklist applies to all work produced during the bootcamp.

---

## Code Submissions

### Every PR Must Include

- [ ] **Working Folder Location**
  - Files in `/working/{frontend|backend}/{name}_{timestamp}/`

- [ ] **README.md**
  - Author name and date
  - Lab/sprint number
  - What was implemented
  - Verification commands run

- [ ] **PLAN.md** (Labs 1+)
  - Approved plan before implementation
  - Files to change listed
  - Steps numbered
  - Risks identified

- [ ] **Tests**
  - Unit tests for new logic
  - Integration tests where applicable
  - All tests passing

- [ ] **Build**
  - Clean build with no errors
  - No new warnings introduced

- [ ] **Copilot Attribution**
  - What AI generated
  - What human verified
  - What human fixed

---

## PR Review Criteria

### Security
- [ ] No secrets in code or comments
- [ ] No production data
- [ ] Placeholders used for sensitive values
- [ ] Input validation present

### Quality
- [ ] Follows repo patterns
- [ ] Self-documenting code
- [ ] Comments for complex logic
- [ ] Error handling present

### Testing
- [ ] Tests cover happy path
- [ ] Tests cover error cases
- [ ] Tests are readable
- [ ] Mocks are appropriate

---

## Sprint Work (Weeks 5-8)

Additional criteria for sprint deliverables:

- [ ] Sprint Packet approved
- [ ] Contract compatibility verified
- [ ] Logging/audit events implemented
- [ ] Performance within targets
- [ ] Security scan clean
- [ ] Accessibility checked (FE)
- [ ] Demo-ready

---

## Gate Passage

To pass any gate, all applicable Definition of Done items must be checked.
