# Daily Dev Block

**Frequency:** Daily (during Sprints 1-2)  
**Duration:** 15 minutes  
**Participants:** All developers

---

## Purpose

Quick sync on:
- **Blockers** that need immediate attention
- **Contract changes** from yesterday
- **PR review load** needing attention

| Type | Resolution Path |
|------|-----------------|
| **Technical** | Pair programming or async help |

---

## PR Review Load Management

If PRs are piling up:
- Assign reviewers explicitly
- Consider PR review time blocks
- Reduce WIP limits

---

## When to Skip

- First 3 days of sprint (focus on planning)
- Last day of sprint (focus on demo prep)
- When team is <4 people and naturally synced
