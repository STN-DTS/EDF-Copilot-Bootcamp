## The 3 Non-Negotiable Rules (Memorize These)

| Rule               | Description                                                      |
| ------------------ | ---------------------------------------------------------------- |
| **1. Plan First**  | Copilot produces a plan (files, steps, tests, risks) before code |
| **2. Small Diffs** | Reviewable increments; no broad refactors                        |
| **3. Proof**       | Build + tests must pass. If it isn't verified, it doesn't ship   |

## Safety Rules (Every Prompt)

**Never include in prompts:**

- Credentials, secrets, tokens, keys
- Production data
- Restricted internal content
- Personal data beyond strict need

**Always start prompts with:**

> "No secrets, no production data. Use placeholders."

**Use placeholders like:** `<TOKEN>`, `<USER_ID>`, `<EMAIL>`, `<ACCOUNT_NUMBER>`, `"SampleCustomerA"`

## Technical Conventions

### Front End

- React 18+ with React Router v7 (RRv7)
- Use RRv7 loader/action conventions
- Normalize API errors to Problem Details shape
- Testing: Vitest + React Testing Library

### Back End

- Spring Boot 3.x REST API
- Layered: controller ‚Üí service ‚Üí repository
- Oracle DB via Spring Data JPA
- Errors as RFC 7807 Problem Details
- Testing: JUnit 5 + Spring Boot Test

### Platform (OpenShift)

- Readiness/liveness probes via Spring Actuator
- Config via env vars, ConfigMap, or Secret
- Resource requests/limits defined
